# AWS MCP Change Log - 2026-04-21

Dokumen ini mencatat perubahan saat penambahan kapabilitas AWS ke MCP lokal Bookcabin.

## Metadata

- Perubahan oleh: `Codex`
- Tanggal: `2026-04-21`
- Konteks: menambahkan observability AWS untuk arsitektur `EC2 + Docker`, diagnostik host lewat SSM dengan command allowlist yang auditable, tail log runtime, metrics, diagnosis delivery path, incident snapshot, drift detection, pipeline/deploy trace, dan fase 7 untuk lookup SHA + remediation sempit

## Ringkasan Perubahan

1. Menambahkan resource MCP baru `awsMcpChangeLog` yang mengarah ke dokumen ini.
2. Menambahkan helper AWS CLI di `mcp/src/index.ts` untuk eksekusi read-only dengan dukungan `AWS_REGION`, `AWS_DEFAULT_REGION`, dan `AWS_PROFILE`.
3. Menambahkan tool `aws_environment_status` untuk mengecek identitas AWS caller, region aktif, versi AWS CLI, dan daftar instance yang dikelola SSM.
4. Menambahkan tool `aws_list_ec2_instances` untuk ringkasan instance EC2, terutama host Docker di EC2.
5. Menambahkan tool `aws_list_ssm_managed_instances` untuk melihat status manajemen host via SSM.
6. Menambahkan tool `aws_list_ecr_repositories` untuk mengecek repository image Docker.
7. Menambahkan tool `aws_list_load_balancers` untuk mengecek ALB/NLB yang aktif.
8. Menambahkan tool `aws_list_cloudwatch_log_groups` untuk observability log group CloudWatch.
9. Menambahkan tool `aws_list_ssm_parameters` dalam mode metadata-only, tanpa pernah membaca nilai parameter.
10. Menambahkan tool `aws_list_cloudfront_distributions` untuk melihat status distribusi CloudFront yang aktif.
11. Menambahkan tool `aws_list_rds_instances` untuk mengecek instance database yang relevan dengan stack Bookcabin.
12. Menambahkan tool `aws_list_target_groups` dan `aws_get_target_group_health` untuk mengecek jalur health check ALB ke host EC2.
13. Menambahkan tool `aws_run_ssm_diagnostic_command` dengan command allowlist observability seperti `docker ps`, `docker compose ps`, `docker logs`, `df -h`, `free -m`, dan `systemctl status docker`.
14. Menambahkan tool `aws_list_cloudwatch_log_streams` untuk memetakan stream log terbaru pada sebuah log group.
15. Menambahkan tool `aws_tail_cloudwatch_logs` untuk membaca event log CloudWatch dalam rentang waktu tertentu, dengan filter pattern opsional.
16. Menambahkan tool `aws_probe_http_endpoint` untuk probe cepat endpoint HTTP/HTTPS dari sisi MCP tanpa harus masuk ke host lebih dulu.
17. Menambahkan tool `aws_list_cloudwatch_metrics` untuk melihat metric yang tersedia per namespace.
18. Menambahkan tool `aws_get_cloudwatch_metric_series` untuk membaca time series metric CloudWatch dengan dimensi yang dipilih.
19. Menambahkan tool `github_actions_workflow_catalog` untuk memetakan workflow GitHub Actions dari file `.github/workflows` saat `gh` CLI tidak tersedia di host.
20. Menambahkan tool `aws_diagnose_delivery_path` untuk diagnosis ringkas jalur `CloudFront -> ALB -> target group -> probe endpoint`.
21. Menambahkan command diagnostik `docker_ps_compact` untuk inventory container yang lebih mudah diparse pada host EC2.
22. Menambahkan helper internal SSM reuse agar tool komposit bisa menjalankan diagnostik host tanpa duplikasi logika polling.
23. Menambahkan tool `aws_list_ecr_image_tags` untuk melihat tag image terbaru pada repository ECR tertentu.
24. Menambahkan tool `aws_detect_ec2_container_drift` untuk membandingkan container yang berjalan di EC2 dengan image terbaru pada repository ECR yang relevan.
25. Menambahkan tool `aws_incident_alb_5xx_snapshot` untuk snapshot cepat insiden 5xx pada ALB: metric sum, target health, dan probe endpoint.
26. Menambahkan helper discovery repo GitHub dari `git remote origin` dan fallback lookup GitHub Actions run via GitHub API bila `gh` CLI tidak tersedia.
27. Menambahkan tool `github_actions_recent_runs` untuk melihat run workflow terbaru dengan sumber `gh` atau GitHub API.
28. Menambahkan tool `aws_incident_target_unhealthy_snapshot` untuk snapshot cepat kasus target unhealthy pada target group.
29. Menambahkan tool `aws_trace_deployment_source_of_truth` untuk menelusuri keterkaitan workflow deploy, runtime container di EC2, dan target health.
30. Menambahkan tool `github_actions_find_run_for_sha` untuk mencari run workflow berdasarkan commit SHA.
31. Menambahkan tool `aws_incident_summary` sebagai pintu masuk komposit per skenario insiden: `alb_5xx`, `target_unhealthy`, dan `deploy_drift`.
32. Menambahkan allowlist remediation yang sangat sempit untuk SSM: `restart_app_container`, `restart_agent_container`, dan `restart_docker_service`.
33. Menambahkan tool `aws_run_ssm_remediation_action` untuk menjalankan remediation allowlist yang auditable tersebut.
34. Memperbarui dokumentasi MCP di `mcp/README.md`, `docs/mcp-server.md`, dan `README.md`.

## Guardrail yang Dipakai

- Semua tool AWS default-nya read-only.
- Tool SSM parameter hanya mengembalikan metadata, bukan value.
- Tool memakai AWS CLI lokal sehingga tetap mengikuti IAM/credential yang sudah ada di environment.
- Region default disetel ke `ap-southeast-1` jika environment variable AWS tidak diisi.
- Tool `aws_run_ssm_diagnostic_command` memang menjalankan perintah di host, tetapi hanya dari allowlist command diagnostik yang tidak dimaksudkan untuk mengubah state sistem.
- Tool `aws_probe_http_endpoint` hanya melakukan request `GET` atau `HEAD`.
- Tool `github_actions_workflow_catalog` hanya membaca file workflow lokal jika `gh` CLI tidak tersedia.
- Tool drift detection menggunakan heuristik nama repository/image/container, jadi hasilnya bersifat bantuan diagnosis dan bukan jaminan sinkronisasi deploy 100%.
- Tool `github_actions_recent_runs` akan memakai GitHub API jika `gh` CLI tidak tersedia, sehingga bisa terpengaruh rate limit atau akses repository private bila token tidak tersedia.
- Tool remediation SSM pada fase 7 adalah write-capable tetapi sengaja dibatasi ke daftar restart action yang kecil dan eksplisit.

## Catatan Operasional

- AWS CLI terdeteksi tersedia saat perubahan ini dibuat.
- Tool baru ini paling berguna untuk investigasi host Docker di EC2, ECR image, ALB, CloudWatch Logs, dan parameter SSM.
- Fase 2 menambah visibilitas CloudFront, RDS, target group health, dan diagnostik host via SSM.
- Fase 3 menambah tail CloudWatch logs, daftar stream log, dan HTTP endpoint probe untuk mempercepat triage insiden.
- Fase 4 menambah metric AWS, katalog workflow GitHub Actions lokal, dan diagnosis delivery path end-to-end.
- Fase 5 menambah drift detection antara ECR dan host EC2 serta incident snapshot untuk kasus ALB 5xx.
- Fase 6 menambah live/degraded workflow run lookup, snapshot target unhealthy, dan trace deploy yang lebih dekat ke source of truth.
- Fase 7 menambah lookup run per SHA, incident summary komposit, dan remediation sempit via SSM.
- Jika agent butuh tindakan write seperti restart instance, deploy container, update parameter, atau command shell bebas via SSM, capability itu harus ditambahkan terpisah dengan audit yang lebih ketat.
